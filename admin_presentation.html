🏸 後端訂單管理系統

為電商平台打造強大的營運後盾


報告人：負責訂單模組的開發者

系統架構

嚴謹的 MVC 與三層式分層設計

核心流程分工

  • View (前端 JSP)：管理員操作介面
  • Controller (Servlet)：指令攔截與資料收集秘書
  • Service (業務邏輯)：核心營運規則與計算大腦
  • DAO (資料存取)：資料庫操作與保險庫管理員
  • Model (Bean)：標準化資料傳遞載體

設計亮點

實踐「單一職責原則」

任務拆分的 Controller

拋棄肥大的單一控制器，將複雜行為細切：

  • 📝 單筆與修改：OrderCreateServlet / OrderUpdateServlet
  • 🔍 清單查詢：OrderListServlet
  • 🚀 進階批次處理：OrderBulkCreateServlet / OrderBulkDeleteServlet

高可讀性、易於擴充維護！

技術挑戰與突破

大量批次處理 (Bulk) 與記憶體綁定

解決痛點

  • 痛點：管理員單筆建檔耗時費力。
  • 方案：實作多筆表單陣列接收。
  • 實作：在 Servlet 使用迴圈，將資料陣列封裝為 List<OrderBean>，再整包交給 Service 處理。
  • 大幅減少資料庫連線浪費，提升效能。

心得與未來展望

  • 體認到 Service 與 DAO 分離的重要性。
  • 純 Servlet 開發過程為未來學習 Spring MVC 打下紮實基礎。
  • 展望：未來可順利轉移至 Spring Boot 框架，並加入更多訂單狀態追蹤！
